# Phase 0 競合再調査

確認日: 2026-05-19

## 目的

Dogfood
Alpha の実装前に、curacuri.ai がどの競合カテゴリと違う価値を出すべきかを整理する。本調査では、機能網羅よりも「自前運用で分類・週報・通知・FAQ候補・限定自動返信を磨き、外部導入前に1週間分の声を5分で把握しつつ低リスクな受付を任せられる体験を作れるか」を基準に比較する。

## 主要競合メモ

### Mava

- サポートリクエスト数ベースで課金し、公開DiscordチャンネルでAI
  Botが回答した質問もリクエストとして数える。
- Web3/コミュニティサポート文脈が強く、問い合わせ削減・チケット処理に寄っている。
- curacuri.ai は「AIが回答する」より先に「運営者が見落とさない週報」を価値の中心に置く。

### CommunityOne

- Discord向けAI回答、分析、コンテンツギャップ検知を前面に出している。
- 汎用のオンラインコミュニティAI基盤として強い。
- curacuri.ai は日本語Discordのニュアンス、導入告知、読み取り専用を初期状態にできる限定自動返信、安全な文言を差別化軸にする。

### GGWP

- AIモデレーションとコミュニティ健全性に強い。
- 感度設定やモデレーター判断からの適応を前提にしている。
- curacuri.ai は toxic 検知や処分ではなく、質問・バグ・要望・不満・称賛を運営判断へ届ける。

### Thena / Pylon

- B2Bサポート、チケット、アカウント文脈、AIエージェントが中心。
- Discord連携はあるが、Slack/Teams/Email/CRMを含むサポート基盤として設計されている。
- curacuri.ai はDogfood Alphaで体験品質を確認し、その後にOSS/self-hostとHosted
  Betaで小規模AIプロダクト公式Discordへ展開する。

### Ticket Tool

- Discord内の問い合わせをチケット化し、フロー、ナレッジベース、データ保持、API、分析を提供する。
- チケット化された会話には強いが、公開チャンネルで流れるVOCの要約には弱い。
- curacuri.ai はチケット化される前の声を拾う。

## Dogfood Alpha に影響する差別化軸

1. 読み取り専用を初期状態にできる限定自動返信設計1で管理者向けの整理と通知に加え、AIキャラクターによる受付・案内・FAQ補助のON/OFF、許可範囲、エスカレーション条件、実行ログまで実装する。
   - 未承認の公式見解、自動BAN、ユーザー評価は初期状態では行わない。
   - 問い合わせ削減と、公式とは分離されたコミュニティ向けインターフェイスを両立する。

2. 日本語Discordの分類と文脈に特化
   - 遠回しな不満、曖昧なバグ報告、内輪ノリ、絵文字、短文質問を拾う。
   - 英語圏サポートツールの翻訳版ではなく、日本語運営の判断に合わせる。

3. 週次運営レポート中心
   - チケット数や返信削減ではなく、「1週間分のDiscordを5分で把握する」体験を最上位に置く。
   - 主要トピック、未回答質問、要望、不満、バグ報告候補、FAQ候補、次アクションをまとめる。

4. 監視感ゼロ設計
   - 「監視」「スコアリング」「危険ユーザー」ではなく、「書記」「見落とし防止」「声を届ける」として導入する。
   - 対象チャンネル、保存期間、DM非対応、分析対象外チャンネルを初期設定から明示する。

5. Dogfood Alphaでの体験品質検証と、Beta以降のHosted導入
   - 初期は自前Discord、疑似ログ、サンプルログで週報・通知・FAQ候補・限定自動返信を磨く。
   - OSS/self-hostはAlphaの主導線にし、Hosted FreeはPublic Beta以降の公開導線にする。
